🤖 AU Agent — Setup Guide

คู่มือ deploy ระบบ LINE → OCR → Dashboard ทีละขั้นตอน | ใช้เวลาประมาณ 45 นาที

📋 สารบัญ

1. GitHub — อัปโหลด code ขึ้น cloud 2. Supabase — สร้างฐานข้อมูล 3. Railway — deploy backend 4. LINE Developers — ตั้ง Webhook 5. Netlify — host dashboard 6. เชื่อม Dashboard กับ Backend 7. ลงทะเบียน LINE Group 8. ทดสอบระบบ
1
GitHub — อัปโหลด Code
Railway ต้องการ GitHub repo เพื่อ deploy อัตโนมัติ
~5 นาที
Step 1.1
สมัคร GitHub (ถ้ายังไม่มี)

เปิดเบราว์เซอร์ไปที่ https://github.com → คลิก Sign up

ใส่ Email, Password, Username → ยืนยัน email → Login เข้ามา

✅ ถ้ามี GitHub account แล้ว ข้ามขั้นตอนนี้ได้เลย
Step 1.2
สร้าง Repository ใหม่

คลิกปุ่ม ＋ New มุมบนขวา หรือไปที่:

https://github.com/new

กรอกข้อมูล:

Repository nameau-agent
VisibilityPrivate (เพื่อความปลอดภัย)
Initialize READMEไม่ต้องติ๊ก

คลิก Create repository

Step 1.3
ติดตั้ง GitHub Desktop (วิธีง่ายที่สุด)

ดาวน์โหลดที่ https://desktop.github.com → ติดตั้ง → Login ด้วย GitHub account

💡 ถ้าใช้ Command Line ได้ → ข้ามไปทำ Step 1.4 แทน
Step 1.4
อัปโหลด folder backend ขึ้น GitHub

วิธีที่ 1 — GitHub Desktop (แนะนำ):

1. เปิด GitHub Desktop → File → Add Local Repository
2. เลือก folder: F:\Claude project\Biomass\Biomass Agent\backend
3. คลิก Publish repository → เลือก repo au-agent → Publish

วิธีที่ 2 — Command Line:

cd "F:\Claude project\Biomass\Biomass Agent\backend" git init git add . git commit -m "Initial commit" git remote add origin https://github.com/YOUR_USERNAME/au-agent.git git push -u origin main
✅ เสร็จแล้ว ไปที่ github.com/YOUR_USERNAME/au-agent จะเห็นไฟล์ทั้งหมด
2
Supabase — สร้างฐานข้อมูล
PostgreSQL ฟรี 500MB พร้อม Realtime
~10 นาที
Step 2.1
สมัครและสร้าง Project

ไปที่ https://supabase.com → คลิก Start your project → Login ด้วย GitHub

คลิก New project → กรอก:

Nameau-agent
Database Passwordตั้งรหัสแข็งแรง บันทึกไว้ด้วย!
RegionSoutheast Asia (Singapore)

คลิก Create new project → รอประมาณ 2 นาที

Step 2.2
รัน SQL Schema สร้างตาราง

เมื่อ project พร้อมแล้ว → ไปที่เมนูซ้าย SQL Editor

คลิก + New query → เปิดไฟล์นี้ด้วย Notepad:

เปิดไฟล์: F:\Claude project\Biomass\Biomass Agent\database\supabase_schema.sql

Copy ทั้งหมด → Paste ลงใน SQL Editor → คลิก Run

✅ จะเห็น "Success. No rows returned" ทุก statement — หมายความว่าตารางถูกสร้างแล้ว
Step 2.3
คัดลอก API Keys

เมนูซ้าย → Settings → API

คัดลอก 2 ค่านี้ เก็บไว้ใช้ใน Railway:

Project URLเช่น https://abcdefgh.supabase.co → นี่คือ SUPABASE_URL
service_role secretคลิก Reveal → Copy → นี่คือ SUPABASE_SERVICE_KEY
⚠️ service_role key มีสิทธิ์เต็ม อย่า share หรือ commit ขึ้น GitHub
3
Railway — Deploy Backend
รัน Node.js server ฟรี — Railway ให้ $5 credit/เดือน
~10 นาที
Step 3.1
สมัครและสร้าง Project

ไปที่ https://railway.app → Login with GitHub

คลิก New Project → เลือก Deploy from GitHub repo

เลือก repo au-agent ที่เพิ่งสร้าง → คลิก Deploy Now

⚠️ ถ้าหา repo ไม่เจอ → คลิก "Configure GitHub App" → Allow access ให้ repo au-agent
Step 3.2
ตั้ง Environment Variables

เข้าไปใน project → คลิก service au-agent → Tab Variables

คลิก + New Variable แล้วเพิ่มทีละตัว:

LINE_CHANNEL_ACCESS_TOKEN = [จาก LINE Developers — ดูขั้นตอนที่ 4] LINE_CHANNEL_SECRET = [จาก LINE Developers — ดูขั้นตอนที่ 4] SUPABASE_URL = [จาก Supabase Step 2.3] SUPABASE_SERVICE_KEY = [จาก Supabase Step 2.3] OCR_MODE = demo PORT = 3000
💡 LINE Token ยังไม่มีตอนนี้ → ใส่ placeholder ก่อนได้ แล้วกลับมาแก้ หลังทำขั้นตอนที่ 4
Step 3.3
ดู Deploy Logs และ URL

Railway จะ build อัตโนมัติ → Tab Deployments → คลิก deployment ล่าสุด → ดู logs

เมื่อเห็น 🚀 AU Agent Backend running on port 3000 แสดงว่าสำเร็จ

ไปที่ Tab Settings → Networking → คลิก Generate Domain

จะได้ URL เช่น:

https://au-agent-production-xxxx.up.railway.app ← เก็บ URL นี้ไว้!
Step 3.4
ทดสอบ Backend

เปิด URL นี้ในเบราว์เซอร์:

https://au-agent-production-xxxx.up.railway.app/health

ต้องเห็น:

{"ok":true,"mode":"demo"}
✅ ถ้าเห็น JSON นี้ → Backend ทำงานปกติแล้ว!
⚠️ ถ้าเห็น Error → ไปดู Logs ใน Railway → มักเกิดจาก env variable ผิด
4
LINE Developers — ตั้ง Webhook
เชื่อม LINE Official Account กับ backend
~5 นาที
Step 4.1
เปิด LINE Developers Console

ไปที่ https://developers.line.biz → Login → เลือก Provider → เลือก Channel ที่มีอยู่

ไปที่ Tab Messaging API

Step 4.2
คัดลอก Token และ Secret
Channel secret อยู่ใน Tab Basic settings → คลิก Copy
Channel access token อยู่ใน Tab Messaging API → ส่วน "Channel access token" → คลิก Issue แล้ว Copy

นำค่าทั้งสองไปใส่ใน Railway Variables (Step 3.2) แล้ว Redeploy

Step 4.3
ตั้ง Webhook URL

Tab Messaging API → หัวข้อ Webhook settings

คลิก Edit → ใส่ URL:

https://au-agent-production-xxxx.up.railway.app/webhook

คลิก Update → แล้วคลิก Verify

✅ ถ้าขึ้น "Success" → Webhook ทำงานแล้ว!
⚠️ ถ้าขึ้น Error → ตรวจสอบว่า Railway URL ถูกต้องและ /health ตอบกลับได้
Step 4.4
ปิด Auto-reply

ใน Tab Messaging API:

Use webhookON ✅
Auto-reply messagesOFF (ไปที่ LINE Official Account Manager แล้วปิด)
Greeting messagesOFF

~2 นาที
Step 5.1
Upload ด้วย Netlify Drop

เปิด https://app.netlify.com/drop ในเบราว์เซอร์

เปิด File Explorer → ไปที่:

F:\Claude project\Biomass\Biomass Agent\

ลาก ไฟล์ planning-mockup.html ไปวางในหน้า Netlify Drop

✅ Netlify จะ upload และให้ URL ภายใน 30 วินาที เช่น https://musical-tarsier-123.netlify.app

บันทึก URL ของ Dashboard

คัดลอก URL ที่ได้ เช่น:

https://musical-tarsier-123.netlify.app ← URL Dashboard ของคุณ

นำ URL นี้ไปใส่ใน Railway Variables:

FRONTEND_URL = https://musical-tarsier-123.netlify.app
💡 ชื่อ Netlify เปลี่ยนได้ที่ Site settings → Change site name
6
เชื่อม Dashboard กับ Backend
แก้ไฟล์ HTML ให้ดึงข้อมูลจาก API จริง
~5 นาที
Step 6.1
เปิดไฟล์ planning-mockup.html

เปิด Notepad++ หรือ VS Code → เปิดไฟล์:

F:\Claude project\Biomass\Biomass Agent\planning-mockup.html

กด Ctrl+F → ค้นหา:

apiStatus
Step 6.2
เพิ่ม API_BASE และ Live Data

หาบรรทัดที่มี apiStatus ในส่วน script → เพิ่ม code นี้ด้านบน:

// ── API CONFIG ──────────────────────────────────── var API_BASE = 'https://au-agent-production-xxxx.up.railway.app'; // แทนที่ด้วย Railway URL ของคุณ // ── HEALTH CHECK ────────────────────────────────── fetch(API_BASE + '/health') .then(function(r){ return r.json(); }) .then(function(d){ var el = document.getElementById('apiStatus'); if(el){ el.textContent = '● Online'; el.style.color = '#22c55e'; } }) .catch(function(){ var el = document.getElementById('apiStatus'); if(el){ el.textContent = '○ Offline'; el.style.color = '#ef4444'; } });

บันทึกไฟล์ → Upload ขึ้น Netlify อีกครั้ง (ลากวางใหม่)

✅ เปิด Dashboard → มุมขวาบนจะเห็น ● Online แทน Offline
7
ลงทะเบียน LINE Group
บอก Bot ว่ากลุ่มไหนคือกลุ่มแผน กลุ่มไหนคือกลุ่มคนขับ
~3 นาที
Step 7.1
เพิ่ม Bot เข้ากลุ่ม LINE

เปิด LINE → กลุ่มที่ต้องการ → กด ชื่อกลุ่ม → เชิญ → ค้นหา Official Account ของคุณ → เพิ่ม

💡 Bot ต้องอยู่ในกลุ่มก่อนจึงจะรับ/ส่งข้อความได้
Step 7.2
ลงทะเบียนกลุ่มแผนขนส่ง

ในกลุ่ม LINE ที่ ผู้จัดการส่งแผนขนส่ง → พิมพ์:

ลงทะเบียน planning กลุ่มแผน Biomass
✅ Bot จะตอบ: "ลงทะเบียนกลุ่ม "กลุ่มแผน Biomass" เป็น planning แล้ว"
Step 7.3
ลงทะเบียนกลุ่มคนขับ (รับรูปใบชั่ง)

ในกลุ่ม LINE ที่ คนขับส่งรูปสลิป → พิมพ์:

ลงทะเบียน driver_upload กลุ่มคนขับ
✅ Bot จะตอบ: "ลงทะเบียนกลุ่ม "กลุ่มคนขับ" เป็น driver_upload แล้ว"
8
ทดสอบระบบ End-to-End
ทดสอบ flow จริงตั้งแต่ LINE ถึง Dashboard
~5 นาที
Step 8.1
ทดสอบแผนขนส่ง

ในกลุ่ม planning → พิมพ์:

วันที่ 20/06/2026 ต้นทาง: โรงงาน ABC ปลายทาง: บ.ยางไทย สินค้า: Wood Pellet ลูกค้า: Biomass Corp รถ 3 คัน

ที่คาดหวัง: Bot ส่ง Flex Card กลับพร้อมปุ่ม ✅ ยืนยัน / 📅 เลื่อน / ❌ ยกเลิก

✅ กด ยืนยัน → ไปดู Dashboard → หน้า แผนขนส่ง → จะเห็นแผนใหม่
Step 8.2
ทดสอบ OCR ใบชั่งน้ำหนัก (Demo Mode)

ในกลุ่ม driver_upload → ส่งรูปภาพใดก็ได้ (Demo Mode จะ simulate OCR)

ที่คาดหวัง: Bot ตอบกลับผล OCR พร้อม Confidence %

✅ Dashboard → บิลทั้งหมด → จะเห็นรายการใหม่เพิ่มขึ้น
💡 Demo Mode ใช้ข้อมูล mock — เปลี่ยน OCR_MODE=claude เมื่อพร้อมใช้จริง
Step 8.3
ทดสอบคำสั่งสรุป

พิมพ์ในกลุ่มใดก็ได้:

สรุป

ที่คาดหวัง: Bot ส่งอีเมลสรุปประจำวัน (ต้องตั้ง SMTP ก่อน)

Step 8.4
เปิดดู Supabase Database

Supabase → Table Editor → เลือก weight_slips หรือ planning

จะเห็นข้อมูลที่ส่งมาจาก LINE แสดงขึ้นมาใน table จริง

✅ ถ้าเห็นข้อมูล → ระบบทำงานครบ loop แล้ว!

🎉 ระบบพร้อมใช้งานแล้ว!

LINE → Webhook → Backend → Supabase → Dashboard

⚙️ BackendRailway URL/health
